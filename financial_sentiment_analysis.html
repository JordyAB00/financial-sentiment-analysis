By: Jordy Alfaro Brenes

This notebook analyzes sentiment in financial texts using a combination of natural language processing and machine learning techniques. The dataset contains financial sentences labeled with sentiment (positive, negative, or neutral).

Dataset Information¶

The dataset combines data from FiQA and Financial PhraseBank, providing financial sentences with sentiment labels. It's intended for advancing financial sentiment analysis research.

Citation: Malo, Pekka, et al. "Good debt or bad debt: Detecting semantic orientations in economic texts." Journal of the Association for Information Science and Technology 65.4 (2014): 782-796.

La gráfica muestra una distribución desbalanceada de sentimientos en el conjunto de datos financieros:

  • Neutral: 3,130 ejemplos (53.7%)
  • Positivo: 1,852 ejemplos (31.8%)

Este desbalance es bastante significativo y típico en datos financieros, donde:

  1. Predominancia neutral: La mayoría de las comunicaciones financieras tienden a ser objetivas y fácticas, evitando lenguaje excesivamente positivo o negativo.

  2. Sesgo positivo: Hay casi el doble de ejemplos positivos que negativos, lo que podría reflejar un optimismo general en las comunicaciones financieras o una tendencia a presentar la información de manera favorable.

  3. Minoría negativa: Los textos con sentimiento negativo representan apenas un 14.5% del total, lo que plantea desafíos para el entrenamiento del modelo, ya que tendrá menos ejemplos para aprender a identificar este tipo de sentimiento.

Implicaciones para el Modelado¶

Este desbalance de clases tendrá varias implicaciones importantes para nuestro modelo de clasificación:

  1. Riesgo de sesgo: El modelo podría sesgarse hacia las clases mayoritarias (neutral y positiva) y tener dificultades para detectar correctamente el sentimiento negativo.

  2. Evaluación del rendimiento: La precisión general (accuracy) podría no ser la mejor métrica para evaluar el modelo debido al desbalance. Será importante analizar métricas como precision, recall y F1-score para cada clase.

  3. Estrategias a considerar:

    • Técnicas de submuestreo o sobremuestreo para balancear las clases
    • Uso de pesos de clase en los algoritmos que lo permitan
    • Ajuste de umbrales de decisión
    • Enfoque en el F1-score como métrica de evaluación

# 4.1 Text Preprocessing Function
def preprocess_text(text):
    """
    Function to preprocess text data
    """
    # Convert to lowercase
    text = text.lower()
    
    # Remove URLs
    text = re.sub(r'http\S+|www\S+|https\S+', '', text, flags=re.MULTILINE)
    
    # Remove mentions and hashtags
    text = re.sub(r'@\w+|\#', '', text)
    
    # Remove punctuation
    text = text.translate(str.maketrans('', '', string.punctuation))
    
    # Remove numbers
    text = re.sub(r'\d+', '', text)
    
    # Tokenize
    tokens = word_tokenize(text)
    
    # Remove stopwords
    stop_words = set(stopwords.words('english'))
    stop_words.update(['s', 't', 've', 'll', 'd', 'm'])  # Add some contractions
    tokens = [word for word in tokens if word not in stop_words and len(word) > 1]
    
    # Lemmatization
    lemmatizer = WordNetLemmatizer()
    tokens = [lemmatizer.lemmatize(word) for word in tokens]
    
    # Join tokens back to text
    preprocessed_text = ' '.join(tokens)
    
    return preprocessed_text

# 5.1 Word Frequency Analysis
def get_top_n_words(corpus, n=20):
    """
    Gets the top n words from a corpus of text
    """
    vec = CountVectorizer().fit(corpus)
    bag_of_words = vec.transform(corpus)
    sum_words = bag_of_words.sum(axis=0) 
    words_freq = [(word, sum_words[0, idx]) for word, idx in vec.vocabulary_.items()]
    words_freq = sorted(words_freq, key=lambda x: x[1], reverse=True)
    return words_freq[:n]

# 7.1 Model Evaluation Function
def evaluate_model(model, X_train, X_test, y_train, y_test, model_name):
    # Train model
    model.fit(X_train, y_train)
    
    # Predictions
    y_pred = model.predict(X_test)
    
    # Evaluation metrics
    accuracy = accuracy_score(y_test, y_pred)
    conf_matrix = confusion_matrix(y_test, y_pred)
    class_report = classification_report(y_test, y_pred)
    
    print(f"--- {model_name} Results ---")
    print(f"Accuracy: {accuracy:.4f}")
    print("\nConfusion Matrix:")
    print(conf_matrix)
    print("\nClassification Report:")
    print(class_report)
    
    # Plot confusion matrix
    plt.figure(figsize=(8, 6))
    sns.heatmap(conf_matrix, annot=True, fmt='d', cmap='Blues',
                xticklabels=sentiment_map.keys(),
                yticklabels=sentiment_map.keys())
    plt.ylabel('Actual')
    plt.xlabel('Predicted')
    plt.title(f'Confusion Matrix - {model_name}')
    plt.tight_layout()
    plt.show()
    
    return model, accuracy

Análisis del Modelo Naive Bayes¶

La matriz de confusión y las métricas de evaluación de Naive Bayes revelan varios aspectos importantes sobre el rendimiento del modelo:

Análisis del Rendimiento¶

  1. Precisión general (Accuracy): 68.35%
  • Un rendimiento moderado que supera la clasificación aleatoria, pero con margen significativo de mejora.
  1. Rendimiento por clase:

  • Clase Negativa (0):

    • Precisión: 64% - Cuando predice negativo, acierta el 64% de las veces
    • Recall: 4% - Sólo identifica el 4% de todos los textos negativos reales
    • F1-score: 8% - Rendimiento extremadamente bajo para esta clase

  • Clase Neutral (1):

    • Precisión: 67% - Precisión moderada para predicciones neutrales
    • Recall: 96% - Excelente capacidad para identificar textos neutrales
    • F1-score: 79% - Buen rendimiento general para esta clase

  • Clase Positiva (2):

    • Precisión: 73% - La mejor precisión entre las tres clases
    • Recall: 52% - Identifica poco más de la mitad de los textos positivos
    • F1-score: 61% - Rendimiento moderado
  1. Observaciones clave de la matriz de confusión:
  • El modelo clasifica la gran mayoría de los textos como neutrales (893 de 1169)
  • Sólo predice la clase negativa 11 veces (menos del 1% de las predicciones)
  • La mayor confusión ocurre entre textos positivos clasificados como neutrales (176 casos)

Problemas Identificados¶

  1. Sesgo extremo hacia la clase neutral: El modelo tiene una fuerte tendencia a clasificar textos como neutrales, lo que explica el alto recall (96%) pero precisión moderada (67%) para esta clase.

  2. Subrepresentación de la clase negativa: El modelo prácticamente ignora la clase negativa, clasificando solo 7 de 172 textos negativos correctamente.

  3. Desbalance en el rendimiento: Hay una diferencia dramática en el recall entre clases (4% vs 96% vs 52%).

Posibles Mejoras¶

  1. Ajuste de la probabilidad previa (class_prior):
  • Modificar las probabilidades a priori para contrarrestar el desbalance de clases
  • Por ejemplo: MultinomialNB(class_prior=[0.33, 0.33, 0.33]) para asignar igual probabilidad a cada clase
  1. Ajuste del umbral de decisión:
  • Implementar un clasificador con umbrales personalizados basados en las probabilidades de Naive Bayes
  • Disminuir el umbral para la clase negativa para aumentar su recall
  1. Técnicas de remuestreo:
  • Sobremuestreo de la clase minoritaria (negativa) usando técnicas como SMOTE
  • Submuestreo de la clase mayoritaria (neutral)
  1. Ingeniería de características:
  • Incorporar bigramas y trigramas para capturar mejor el contexto (por ejemplo, "no good" vs "good")
  • Experimentar con esquemas de pesaje como TF-IDF con diferentes parámetros
  • Añadir características basadas en léxicos de sentimiento financiero
  1. Considerar modelos alternativos:
  • Aunque Naive Bayes es eficiente, otros modelos como SVM o Random Forest podrían manejar mejor el desbalance
  • Ensemble de múltiples clasificadores para mejorar el rendimiento general
  1. Técnicas específicas para Naive Bayes:
  • Ajustar el parámetro alpha (suavizado de Laplace) para encontrar el valor óptimo
  • Probar diferentes variantes de Naive Bayes (Bernoulli NB en lugar de Multinomial)

El principal desafío es mejorar la detección de la clase negativa sin sacrificar demasiado el rendimiento en las otras clases. Un enfoque combinado de ajuste de parámetros y reequilibrio de clases probablemente dará los mejores resultados.

  1. Mayor división de clases: Random Forest intenta equilibrar más las predicciones entre las tres clases, lo que reduce su precisión general pero potencialmente ofrece una visión más matizada.

  1. Posible sobreajuste: Random Forest puede estar sobreajustándose a características específicas del conjunto de entrenamiento que no generalizan bien.

  2. Características ruidosas: El modelo podría estar dando importancia a palabras o patrones que no son verdaderamente discriminativos para el sentimiento.

  3. Desbalance de clases: A pesar de su capacidad para manejar datos desbalanceados, este Random Forest parece afectado por el desbalance, especialmente en la clase negativa.

  1. Ajuste de hiperparámetros:
  • Modificar max_depth para controlar el sobreajuste
  • Aumentar n_estimators para mejorar la generalización
  • Ajustar min_samples_leaf para evitar divisiones demasiado específicas
  1. Ponderación de clases:
  • Implementar class_weight='balanced' o pesos personalizados
  • Experimentar con técnicas de submuestreo integradas en el algoritmo
  1. Selección de características:
  • Utilizar la importancia de características de Random Forest para eliminar características ruidosas
  • Implementar selección de características recursiva
  1. Ensemble con otros modelos:
  • Combinar las predicciones de Random Forest con Regresión Logística mediante votación o stacking
  • Usar Random Forest para generar características para un modelo final de Regresión Logística

Conclusión¶

Random Forest muestra un comportamiento interesante al intentar equilibrar más las predicciones entre clases, pero su rendimiento general es inferior a Regresión Logística. Sin embargo, podría ser valioso como parte de un ensemble de modelos o después de un ajuste más exhaustivo de hiperparámetros.

Basado en los resultados actuales, la Regresión Logística sigue siendo el modelo más efectivo entre los tres evaluados, ofreciendo el mejor equilibrio entre precisión general y rendimiento para cada clase.

# 10.1 Sample Prediction Function
def predict_sentiment(text, model, vectorizer, sentiment_map_inv):
    # Preprocess the text
    cleaned = preprocess_text(text)
    
    # Vectorize
    text_vectorized = vectorizer.transform([cleaned])
    
    # Predict
    prediction = model.predict(text_vectorized)[0]
    proba = model.predict_proba(text_vectorized)[0]
    
    # Map back to sentiment label
    sentiment = sentiment_map_inv[prediction]
    
    return sentiment, proba

# Create inverse mapping
sentiment_map_inv = {v: k for k, v in sentiment_map.items()}
